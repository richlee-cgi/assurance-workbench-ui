User Guide

Local only

What this tool does

Assure-O-Matic 3000 Workbench is a local browser interface for assurance-cli. It helps configure evidence-pack runs, preview the command, save results into a Workbench evidence folder, and inspect completed runs.

The UI does not replace the CLI. The CLI remains the evidence engine.

Settings

Use Settings to point the UI at the local assurance executable, choose the evidence output folder, and set default Confluence, Jira, Azure and repository scope.

Settings are saved locally in an ignored file. API tokens and credentials are not stored in this UI.

Topics and presets

The topic is passed to selected evidence sources as the main search text. Presets provide common assurance search shapes and inclusion defaults.

  • architecture: design decisions, integrations and dependencies.
  • delivery: implementation tickets, status, releases and blockers.
  • operations: runtime, deployment, monitoring and operational readiness.
  • dataverse: Power Platform, solutions, connectors and connection references.
  • performance: APIM, Functions, scaling, capacity and timeouts.
  • risk: known risks, blockers, incidents, defects, warnings and mitigations.

Sources

Confluence and Jira are queried by default when selected. Azure adds bounded Resource Graph evidence. Dataverse adds a Power Platform snapshot from the current pac profile.

Code repositories add local Git evidence from selected repo roots and repositories.

Search exclusions

Settings can exclude Confluence pages under configured parent page IDs or URLs, and Jira issues assigned to configured teams.

Use exclusions for known governance or assurance areas that otherwise pollute product evidence. Exclusions are passed to assurance-cli and shown in the evidence pack so the scope remains auditable.

Code and GitHub

Code repository evidence prefers local clones. It can include matching file lines, recent commit messages and repository metadata.

Include PR metadata when Jira links or local evidence should be connected to pull requests. Include bounded diffs only when a limited PR diff is useful evidence.

GitHub fallback allows read-only lookup through gh when PR metadata cannot be resolved locally. It requires local GitHub CLI authentication.

Cache options

By default, Atlassian responses are cached by assurance-cli and reused where suitable.

Refresh cache bypasses existing entries for the run and updates the cache with fresh responses. No cache disables cache reads and writes for the run.

Running evidence packs

The command preview shows the CLI command that will run. The output folder preview shows where the next run will be saved.

Start a run from the main page. Long-running runs can be cancelled while they are in progress.

Results

The Results page lists saved evidence packs. Use filters to narrow by topic, preset or source.

Each result shows metadata, source coverage, command, warnings, logs and links to saved files. Completed runs can be re-run from saved request metadata.

Troubleshooting

Use Check assurance CLI, Check Azure and Check Dataverse on the Settings page to verify local tooling.

If GitHub fallback or PR evidence fails, check that gh auth status works in the same environment that runs the UI.
